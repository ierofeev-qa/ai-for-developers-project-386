# Calendar Booking Frontend

Фронтенд приложения для бронирования календаря, разработанный на TypeScript + Vite + Mantine.

## Требования

- Node.js 18+
- npm или yarn

## Установка

```bash
npm install
```

## Запуск

### Вариант 1: Запуск с мок-сервером Prism (рекомендуется для разработки)

Этот вариант запускает одновременно мок-сервер API на порту 3001 и фронтенд на порту 5173:

```bash
npm run dev:mock
```

### Вариант 2: Отдельный запуск

Запустите мок-сервер API:
```bash
npm run mock
```

В другом терминале запустите фронтенд:
```bash
npm run dev
```

### Вариант 3: Запуск с реальным бэкендом

Если у вас запущен реальный бэкенд, просто запустите фронтенд:

```bash
npm run dev
```

При необходимости измените URL API в файле `.env`:
```
VITE_API_URL=http://localhost:3001
```

## Структура проекта

```
src/
├── api/              # API клиент и функции
├── components/
│   ├── admin/        # Компоненты админ-панели
│   ├── common/       # Общие компоненты
│   └── guest/        # Компоненты для гостей
├── hooks/            # TanStack Query хуки
├── mocks/            # Prism мок-данные
├── pages/            # Страницы приложения
├── providers/        # React Context провайдеры
├── types/            # TypeScript типы
└── App.tsx           # Главный компонент
```

## Функциональность

### Для гостей:
- Просмотр списка типов событий
- Выбор времени из доступных слотов (14 дней)
- Создание бронирования без регистрации
- Просмотр подтверждения бронирования

### Для владельца (админ-панель):
- Просмотр предстоящих встреч
- Управление типами событий (CRUD)
- Отмена бронирований

## Технологии

- **Vite** — сборка и dev-сервер
- **React 18** — UI фреймворк
- **TypeScript** — типизация
- **Mantine** — UI библиотека
- **TanStack Query** — работа с API и кеширование
- **Axios** — HTTP клиент
- **Day.js** — работа с датами
- **Prism** — мокирование API для разработки

## Скрипты

- `npm run dev` — запуск dev-сервера
- `npm run dev:mock` — запуск dev-сервера с мок-сервером
- `npm run mock` — запуск только мок-сервера
- `npm run build` — сборка для продакшена
- `npm run preview` — предпросмотр сборки
- `npm run lint` — проверка кода линтером
