# Playwright E2E Tests

Этот проект содержит End-to-End тесты на Playwright для тестирования основных пользовательских сценариев системы бронирования встреч.

## Структура тестов

```
frontend/tests/
├── guest-booking.spec.ts      # Сценарии бронирования гостем
└── admin-management.spec.ts   # Сценарии управления администратором
```

## Сценарии тестирования

### 1. Guest Booking Flow (`guest-booking.spec.ts`)

**Основной тест:** `guest can successfully book a meeting`

Покрывает полный сценарий бронирования:
1. Открытие главной страницы со списком типов событий
2. Выбор типа события для бронирования
3. Выбор доступной даты и временного слота
4. Заполнение формы данными гостя (имя, email, телефон, заметки)
5. Отправка формы бронирования
6. Проверка страницы подтверждения с деталями бронирования
7. Возврат на главную страницу

**Дополнительные тесты:**
- `guest can navigate back from booking flow` - проверка навигации "Назад"

### 2. Admin Management Flow (`admin-management.spec.ts`)

**Основной тест:** `admin can create and delete event types`

Покрывает управление типами событий:
1. Переход на страницу входа в админ-панель
2. Вход как владелец (без авторизации)
3. Переход на вкладку "Типы событий"
4. Создание нового типа события
5. Проверка появления в списке
6. Удаление созданного типа события
7. Проверка удаления из списка

**Дополнительные тесты:**
- `admin can cancel event type deletion` - отмена удаления
- `admin can navigate between tabs` - навигация между вкладками
- `admin can log out` - выход из админ-панели

## Запуск тестов

### Локальный запуск

```bash
# Перейти в директорию frontend
cd frontend

# Установить зависимости
npm ci

# Установить браузеры Playwright
npx playwright install chromium

# Запустить все тесты
npm run test:e2e

# Запустить в UI режиме (для отладки)
npm run test:e2e:ui

# Запустить в debug режиме
npm run test:e2e:debug

# Просмотреть отчет
npm run test:e2e:report
```

### Запуск в CI

Тесты автоматически запускаются в GitHub Actions при каждом push или pull request:
- Workflow: `.github/workflows/playwright.yml`
- Используется только Chromium (без Firefox и WebKit)
- Backend и Frontend автоматически запускаются перед тестами
- В конце загружаются артефакты с отчетами

## Конфигурация

**`playwright.config.ts`:**
- Базовый URL: `http://localhost:5173`
- Браузер: Chromium только
- WebServer: Автоматически запускает backend (порт 3001) и frontend (порт 5173)
- Retries: 2 попытки в CI, 0 локально
- Workers: 1 в CI (последовательно), undefined локально (параллельно)

## Data-testid атрибуты

Для стабильной работы тестов используются `data-testid` атрибуты на ключевых элементах UI:

### Главная страница
- `event-types-list-page` - контейнер страницы
- `page-title` - заголовок страницы
- `event-types-grid` - сетка карточек событий
- `event-type-card-{id}` - карточка конкретного события
- `book-button` - кнопка "Записаться"
- `admin-panel-button` - кнопка перехода в админку

### Выбор времени
- `time-slot-selector` - контейнер выбора времени
- `date-picker` - календарь выбора даты
- `time-slot-{time}` - конкретный временной слот (например, `time-slot-09:00`)
- `back-button` - кнопка "Назад"

### Форма бронирования
- `booking-form` - контейнер формы
- `name-input` - поле ввода имени
- `email-input` - поле ввода email
- `phone-input` - поле ввода телефона
- `notes-input` - поле заметок
- `submit-booking-button` - кнопка подтверждения

### Подтверждение бронирования
- `booking-confirmation-page` - страница подтверждения
- `confirmation-title` - заголовок "Запись подтверждена!"
- `go-home-button` - кнопка возврата на главную

### Админ-панель
- `admin-login-page` - страница входа
- `login-button` - кнопка входа
- `admin-page` - контейнер админ-панели
- `event-types-tab` - вкладка "Типы событий"
- `meetings-tab` - вкладка "Встречи"
- `create-event-type-button` - кнопка создания типа события
- `event-types-table` - таблица типов событий
- `delete-event-type-{id}` - кнопка удаления
- `confirm-delete-button` - кнопка подтверждения удаления
- `event-type-modal` - модальное окно формы
- `event-type-name-input` - поле названия типа
- `event-type-duration-input` - поле длительности

## Отчеты

После запуска тестов генерируется HTML-отчет:
- Локально: `playwright-report/index.html`
- В CI: Загружается как артефакт `playwright-report`

Отчет содержит:
- Скриншоты при падении тестов
- Трейсы выполнения
- Видео тестов (опционально)
- Подробные логи

## Примечания

1. Тесты используют автоматический запуск серверов (webServer в конфиге)
2. Время ожидания загрузки слотов - до 500мс для стабильности
3. Выходные дни (суббота, воскресенье) пропускаются автоматически
4. Тесты создают и удаляют тестовые данные, не оставляя мусора
